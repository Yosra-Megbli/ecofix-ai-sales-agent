# Ecofix AI Sales Agent CRM Structure

## Purpose

This document defines the CRM data structure used by the Ecofix AI Sales Agent.

The objective is to:
- Store prospect information.
- Track conversation history.
- Measure sales performance.
- Avoid duplicate leads.
- Analyze conversion performance.

The CRM must become the central source of sales information.

---

# CRM Architecture

The CRM contains three main entities:
1. Leads
2. Conversations
3. Campaigns

---

# Table 1: Leads

## Purpose
Stores all prospects contacted by the AI agent.

---

## Lead Identification

### lead_id
Type: Unique identifier  
Example: `LEAD_000001`

### created_at
Type: Datetime  
Purpose: Track lead creation time.

### source
Type: String  
Examples:
- Cold CSV file
- Website
- Meta Ads
- Referral
- Campaign

---

# Customer Information

## first_name
Type: String  
Customer first name.

## last_name
Type: String  
Customer last name.

## phone
Type: String  
Customer phone number.

## email
Type: String  
Customer email address.

## address
Type: String  
Customer address.

## birth_date
Type: Date  
Required only if validated by Ecofix process.

---

# Energy Information

## current_supplier
Type: String  
Current energy supplier.

## energy_type
Type: Enum  
Values:
- Electricity
- Gas
- Both

## ean_number
Type: String  
Energy connection identifier.

---

# Qualification Information

## lead_score
Type: Integer  
Range: 0-100  
Calculated by the scoring engine.

## qualification_status
Type: Enum  
Values:
- NEW
- CONTACTED
- RESPONDED
- QUALIFIED
- FOLLOW_UP
- NOT_INTERESTED
- CONVERTED

## interest_level
Type: Enum  
Values:
- HIGH
- MEDIUM
- LOW

## motivation
Type: String  
Examples:
- Looking for better offer
- Wants comparison
- Unhappy with current supplier
- Interested in digital management

## qualification_reason
Type: Text  
Example: `"Customer interested in switching and provided EAN number."`

---

# Table 2: Conversations

## Purpose
Stores all AI/customer interactions.

---

## conversation_id
Unique identifier.

## lead_id
Relation: Leads table

## started_at
Datetime

## ended_at
Datetime

## channel
Type: Enum  
Values:
- Web Chat
- Telegram
- WhatsApp
- Voice

## messages
Type: JSON  
Example:
```json
[
 {
  "role":"user",
  "content":"Je suis intéressé"
 },
 {
  "role":"assistant",
  "content":"Merci..."
 }
]
```

## conversation_summary
Text summary generated by AI.  
Example: `"Customer currently with Engie, interested in comparing offers."`

## detected_intents
Array:  
Examples:
* Interested
* Price question
* Objection
* Supplier comparison

## objections_detected
Array:  
Examples:
* Already has supplier
* Needs time
* Price concern

---

# Table 3: Campaigns

## Purpose
Track marketing campaigns and lead sources.

---

## campaign_id
Unique identifier.

## campaign_name
Example: `Cold Leads Belgium January`

## source
Examples:
* CSV Import
* Meta Ads
* Website

## total_contacts
Number of leads contacted.

## responses
Number of responses.

## qualified_leads
Number of qualified leads.

## conversions
Number of sales.

---

# Event Tracking

Create an event log.  
Table: `lead_events`

---

Fields:

## event_id
Unique identifier.

## lead_id
Relation.

## event_type
Values:
* LEAD_CREATED
* MESSAGE_SENT
* CUSTOMER_REPLIED
* QUALIFIED
* FOLLOW_UP_CREATED
* CONVERTED
* REJECTED

## timestamp
Datetime.

## metadata
Type: JSON (additional information)  
Example:
```json
{
  "score": 85,
  "reason": "Interested in switching"
}
```

# Analytics Enabled

With this structure the dashboard can calculate:

## Contact Rate
How many leads were contacted.

## Response Rate
Responses / contacted leads.

## Qualification Rate
Qualified leads / responses.

## Conversion Rate
Sales / qualified leads.

## Cost Per Acquisition
Campaign cost / conversions.
